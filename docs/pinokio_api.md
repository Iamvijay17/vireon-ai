# 1. Switch TTS Model

Change the active TTS model.

### Endpoint

```
POST /switch_tts_model
```

### Request

```json
{
  "new_choice": "F5-TTS_v1"
}
```

### Response

```json
[
  "model_path",
  "vocab_path",
  "config"
]
```

---

# 2. Basic TTS

Generate speech using a reference speaker.

### Endpoint

```
POST /basic_tts
```

### Parameters

| Name | Type | Required |
|------|------|----------|
| ref_audio_input | Audio File | ✅ |
| ref_text_input | string | ✅ |
| gen_text_input | string | ✅ |
| remove_silence | boolean | ❌ |
| randomize_seed | boolean | ❌ |
| seed_input | number | ❌ |
| cross_fade_duration_slider | number | ❌ |
| nfe_slider | number | ❌ |
| speed_slider | number | ❌ |

### Example Request

```json
{
  "ref_audio_input": "<audio>",
  "ref_text_input": "Hello everyone.",
  "gen_text_input": "Welcome to today's podcast.",
  "remove_silence": false,
  "randomize_seed": true,
  "seed_input": 0,
  "cross_fade_duration_slider": 0.15,
  "nfe_slider": 32,
  "speed_slider": 1.0
}
```

### Response

```json
[
  "<generated_audio>",
  "<spectrogram>",
  "<reference_text>",
  12345
]
```

# 3. MultiSpeech

Generate multi-speaker conversations using multiple reference voices.

### Endpoint

```
POST /multispeech
```

### Example Request

```json
{
  "script": "{Speaker1} Hello!\n{Speaker2} Hi!",
  "voices": [
    {
      "speaker": "Speaker1",
      "audio": "<speaker1.wav>"
    },
    {
      "speaker": "Speaker2",
      "audio": "<speaker2.wav>"
    }
  ]
}
```

### Response

```json
{
  "audio": "<conversation_audio>"
}
```

---

# 4. Speech Edit

Edit or regenerate portions of an existing speech.

### Endpoint

```
POST /speech_edit
```

### Input

- Existing audio
- New text
- Reference speaker

### Response

```json
{
  "audio": "<edited_audio>"
}
```

# 5. Set Custom Model

Load a custom checkpoint.

### Endpoint

```
POST /set_custom_model
```

### Request

```json
{
  "custom_ckpt_path": "...",
  "custom_vocab_path": "...",
  "custom_model_cfg": "{...}"
}
```

# 6. Load Text From File

Load text directly from a `.txt` file.

### Endpoint

```
POST /load_text_from_file
```

### Content Type

```
multipart/form-data
```

### Form Data

| Name | Type |
|------|------|
| file | Text File |

### Response

```json
"Loaded text..."
```

# Recommended APIs

| Endpoint | Purpose |
|----------|---------|
| `/switch_tts_model` | Select active TTS model |
| `/basic_tts` | Single-speaker voice cloning |
| `/multispeech` | Multi-speaker conversation generation |
| `/speech_edit` | Edit generated speech |
| `/set_custom_model` | Load custom model checkpoints |
| `/load_text_from_file` | Load text from a file |

---

# APIs You Can Ignore

These endpoints are automatically generated by Gradio for the web interface and are generally **not required** for backend integrations.

```
/lambda
/lambda_1
/lambda_2
...
/lambda_n

/load_text_from_file_1
/load_text_from_file_2
...
/load_text_from_file_n

/collapse_accordion
```

These helper endpoints do not perform speech synthesis and can safely be ignored in most production applications.
